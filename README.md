# 台股每日監控機器人（免費架構）

## 架構
GitHub Actions（排程）→ 證交所 OpenAPI（免費資料源）→ pandas 運算與篩選
→ LINE Messaging API / Telegram Bot 推播 → CSV 存回 repo 累積歷史

## 使用步驟

### 1. 建立 GitHub repo
把 `monitor.py`、`.github/workflows/daily_monitor.yml` 放進你的 repo。

### 2. 申請 LINE Messaging API（免費，每月200則）
1. 到 [LINE Developers](https://developers.line.biz/) 建立 Provider 與 Channel（Messaging API 類型）
2. 取得 `Channel Access Token`
3. 取得推播對象 ID：
   - 個人：加官方帳號好友後，用 Webhook 或 LINE Developers 的測試工具取得 `userId`
   - 群組：把官方帳號拉進群組（需先在官方帳號設定開啟「允許被邀請加入群組」），透過 Webhook 事件取得 `groupId`

### 3. 申請 Telegram Bot（免費，無則數上限，作為備援）
1. 在 Telegram 搜尋 `@BotFather`，用 `/newbot` 建立機器人，取得 `Bot Token`
2. 把機器人加入你的聊天或群組，用 `https://api.telegram.org/bot<TOKEN>/getUpdates` 取得 `chat_id`

### 4. 在 GitHub repo 設定 Secrets
到 repo 的 `Settings > Secrets and variables > Actions`，新增：
- `LINE_CHANNEL_ACCESS_TOKEN`
- `LINE_TARGET_ID`
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`

### 5. 測試
到 GitHub repo 的 Actions 頁籤，手動觸發 `台股每日監控` workflow（workflow_dispatch），
確認 LINE / Telegram 有收到訊息，且 `docs/data/` 資料夾有新的 CSV 檔案。

### 6. 開啟 GitHub Pages 儀表板（手機也能看的網頁）

1. 確認 repo 裡有 `docs/index.html`（本次新增的儀表板網頁）
2. 到 repo 的 `Settings > Pages`
3. **Source** 選擇 `Deploy from a branch`，**Branch** 選 `main`，資料夾選 `/docs`，儲存
4. 等 1-2 分鐘後，會出現一個網址，例如 `https://你的帳號.github.io/repo名稱/`
5. 用手機瀏覽器打開這個網址，可以「加入主畫面」，體驗上就像一個 App 圖示

儀表板會自動讀取 `docs/data/latest.csv`（由 `monitor.py` 每次執行時產生），
畫面包含：頂部跑馬燈、核心自選股卡片、動態觀察清單分類清單。
如果還沒有真實資料，畫面會先顯示示範資料並提示尚未偵測到檔案，這是正常現象。

## 重大更新：資料路徑修正 + K線圖 + 關注股票機制

**⚠️ 重要修正**：之前版本 `monitor.py` 把資料存到 `data/` 資料夾，但 GitHub Pages
只公開發布 `docs/` 資料夾，導致網頁其實抓不到真實資料。這次已修正，所有資料改存到
**`docs/data/`** 底下。如果你的 repo 裡還有舊的 `data/` 資料夾，可以直接刪除，不影響運作。

### 新增功能

1. **歷史資料保留半年**：`docs/data/history.json` 現在保留約130個交易日（近半年）的
   OHLC（開高低收）資料，並自動計算 MA5、MA20、KD 指標
2. **K線圖查詢**：網頁新增「個股歷史走勢」區塊，用免費開源套件 lightweight-charts
   畫出蠟燭圖，疊加 MA5/MA20 均線，並顯示最新 K/D 數值
3. **關注股票管理（免費的網頁互動機制）**：
   - 網頁上輸入股票代號、按「送出關注請求」，會開一個**預先填好內容的 GitHub Issue**頁面
   - 你按下 GitHub 的「Submit new issue」送出後，下一次排程執行時，
     `monitor.py` 會自動讀取這個 Issue、把股票加入監控清單、開始推播提醒，並自動關閉該 Issue
   - 這個機制完全免費，不需要架設額外的伺服器，也不會讓任何金鑰暴露在網頁上

### 這個機制的限制（誠實說明）

- 不是「按下去馬上推播」，而是「送出請求 → 排程下次執行時生效」，最長要等到下一個交易日下午2:15
- K線圖只有「核心自選股」和「已核准的關注股票」才有歷史資料，動態觀察清單（價格異常等）
  因為每天篩選出來的股票都不同，沒有持續累積歷史，所以不會出現在K線圖選單裡

## 這次新增：還原股價（除權息調整） + 成交量確認

### 還原股價

參考證交所公告的除權息計算公式：
```
除權息參考價 = (除權息前收盤價 - 現金股利) / (1 + 無償配股率)
```
系統現在會：
1. **每日執行時**，檢查當天是不是監控股票的除權息日，如果是，自動把歷史資料往回調整
2. **一次性回補**（`backfill_history.py`）時，也會抓取回補區間內的除權息事件，一併套用調整

調整後的價格只影響「用來計算技術指標的歷史資料」，**不影響今天的實際股價顯示**——
最新一天的價格永遠是真實股價，只有「過去」的價格會被回溯校正，目的是避免除權息當天
的價格跳空被誤判成假的MA/KD交叉訊號。

### ⚠️ 這個功能是「盡力而為」，需要你幫忙驗證

除權息資料用的證交所端點（`TWT49U`）沒有正式的欄位文件，程式用「欄位名稱關鍵字比對」
的方式盡量抓對資料，如果格式跟預期不同，**程式會自動跳過調整、安靜地退回原始價格**，
不會讓系統掛掉，但也代表這個功能可能沒有實際生效。

**驗證方式**：找一檔你知道最近除過權息的股票（例如某檔剛發放股利的個股），
看它在除權息日前後的K線圖，如果價格連續、沒有出現不合理的斷崖式跳空，
代表還原股價有正常運作；如果還是看到跳空，代表這個功能暫時沒抓對資料，
可以跟我說，我再幫你排查。

### 成交量確認

每個買賣訊號後面現在會多一句提示：
- **「有放量確認」**：當天成交量達到近5日均量的1.5倍以上，代表訊號有量能支撐，可信度較高
- **「量能未明顯放大」**：成交量沒有明顯增加，訊號本身依然有效，但可信度相對較低，建議謹慎看待

這個1.5倍的門檻是技術分析教學中常見的判斷標準（不同資料來源會用1.5~2倍不等），
如果你想調整這個倍數，可以改 `monitor.py` 裡的 `VOLUME_CONFIRM_MULTIPLIER` 這個變數。

## 這次新增：盤中買賣訊號偵測

有了半年的歷史資料後，盤中即時更新（10:00 / 12:00）現在會多做一件事：
把「當下即時股價」當作假設的今日收盤價，重新算一次MA5/MA20/KD，
跟「昨天收盤算出來的數值」比對，看有沒有觸發交叉或突破。

### ⚠️ 這是「暫定」訊號，不是最終結果

盤中算出來的訊號，因為股價在收盤前還可能變動，**跟收盤後正式算出來的結果不一定一樣**，
推播訊息跟網頁上都會清楚標示「盤中暫定」字樣，提醒你這只是提早預告、還沒定案，
真正的訊號要等收盤後那次（`daily_monitor.yml`）才算數。

### 資料存放位置

盤中訊號存在 `docs/data/intraday_signals.json`，跟收盤後的 `signals.json` 是分開的檔案，
避免互相覆蓋。網頁會自動把兩份合併顯示在核心自選股卡片上。

## 一次性回補歷史K線資料（不用等系統慢慢累積）

新增了 `backfill_history.py`，直接用證交所「個股日成交資訊」API 把過去**半年**的
完整開高低收(OHLC)抓回來，灌進 `docs/data/history.json`，這樣**現在馬上**就能有
K線圖、MA5/MA20、KD這些技術指標，不用等系統一天一天累積滿一個月。

### 使用方式（只需要跑一次）

1. 上傳 `backfill_history.py` 到 repo 根目錄，以及 `.github/workflows/backfill.yml`
2. 到 GitHub 的 **Actions** 頁籤，找到「**一次性回補歷史K線資料**」這個workflow
3. 點 **Run workflow** 手動觸發（這個排程**只能手動觸發，不會每天自動跑**，跑一次就夠了）
4. 跑完後，`docs/data/history.json` 就會有過去半年的真實資料，K線圖跟MA/KD訊號馬上就能顯示

### 限制

- **系統電(5309)是上櫃股票，這支API只查得到上市股票的資料，所以系統電這次還是回補不到**，
  程式會印出提示訊息並跳過，不影響其他9檔股票的回補
- 回補完成後，之後 `monitor.py` 每天正常執行會接著往後累積，兩者是同一份 `history.json`，
  資料會自然銜接，不需要額外處理
- 之後不需要再跑這支程式，只有系統剛上線、或你新增了一批新的關注股票想馬上有歷史資料時才需要用到

## 這次新增：盤中即時更新（一天3次：10:00 / 12:00 / 收盤後）

### 新增檔案

`.github/workflows/intraday_monitor.yml`：獨立排程，台灣時間 **10:00 與 12:00** 各執行一次，
加上原本收盤後的 `daily_monitor.yml`（約14:15），**一天總共推播3次**。

### 涵蓋範圍（重要限制）

- **會即時更新**：核心自選股 + 你透過網頁申請的關注股票，這些股票的價格會在10:00、12:00更新
- **不會即時更新**：價格異常、成交量異常、熱度潛力股這幾個動態分類，維持一天更新一次（收盤後）。
  原因是這些分類需要「全市場約2000檔股票」的資料做排序篩選，而免費的盤中即時報價只能一檔一檔查，
  沒有全市場即時打包資料可用，只有收盤後才有整批下載的資料源

### 免費額度確認

- **GitHub Actions**：因為 repo 是 Public（公開），Actions 完全免費、沒有時數限制，這個排程頻率完全不用擔心
- **LINE推播**：一天3次 × 約21個交易日 ≈ 63則/月，遠低於LINE免費的200則上限，額度非常寬裕

### 盤中報價資料源的限制

盤中即時報價使用的是證交所「基本市況報導網站」的內部端點，**不是正式文件化的官方OpenAPI**，
穩定性沒有保障，如果證交所調整這個端點格式，盤中更新可能會失效（收盤後的每日更新不受影響，
因為那個用的是正式的OpenAPI）。如果之後盤中更新持續失敗，這是最可能的原因。

### LINE推播格式精簡

- 【價格異常】【成交量異常】【熱度潛力股】這幾個動態分類，原本只列代號，現在改成：
  顯示前5檔（而非全部10檔），每檔附上股票名稱、收盤價、漲跌幅
- 網頁端則維持顯示完整10檔，不受這個精簡影響——**手機推播和網頁使用同一份底層資料**，

原本【價格異常】等分類是純文字清單，現在跟核心自選股一樣改成卡片，每張卡片包含：
代號、名稱、股價、漲幅、走勢圖（sparkline）、成交量。

**走勢圖的限制**：動態分類的股票是每天篩選出來的，不像核心自選股會持續追蹤，
所以剛開始這些卡片的走勢圖大多會顯示「尚無多日歷史資料」。系統現在已經改成：
只要一檔股票有進入過任何動態分類，就會開始記錄它的歷史價格，如果之後這檔股票
連續好幾天都被篩選出來，走勢圖就會逐漸累積出線條；如果只出現一天就再也沒出現，
就會一直維持只有一個資料點的狀態，這是資料本質上的限制，不是程式錯誤。

## 這次新增：K線型態辨識（錘子/吊人/吞噬/晨星暮星 + 簡化版雙重頂底）

### 新增判斷的型態

| 型態 | 判斷方式 | 意義 |
|---|---|---|
| 錘子線 | 下跌趨勢中，長下影線＋小實體 | 偏多參考 |
| 吊人線 | 上漲趨勢中，長下影線＋小實體（跟錘子線同形狀） | 偏空參考 |
| 多頭吞噬 | 下跌後，紅K實體完全包覆前一根黑K實體 | 偏多參考 |
| 空頭吞噬 | 上漲後，黑K實體完全包覆前一根紅K實體 | 偏空參考 |
| 晨星 | 長黑K→小實體跳空→長紅K收深入第一根實體 | 偏多參考 |
| 暮星 | 長紅K→小實體跳空→長黑K收深入第一根實體 | 偏空參考 |
| M頭/雙重頂 | 簡化判斷：近期兩個高度相近的高點 | 偏空參考，準確度較低 |
| W底/雙重底 | 簡化判斷：近期兩個高度相近的低點 | 偏多參考，準確度較低 |

這些型態的定義參考了多篇公開技術分析教學（量化通、股澐、Investopedia等資料來源），
會自動併入原本的買賣訊號清單，一起顯示在LINE推播跟網頁核心自選股卡片上。

### 沒有做「頭肩頂/頭肩底」

這個型態需要判斷「三個轉折點」的複雜圖形（左肩、頭、右肩），演算法上比雙重頂/底更主觀、
更容易誤判，這次先不自動化，如果你之後想要，我們可以再另外討論怎麼做比較穩健。

### 「M頭/雙重頂」「W底/雙重底」是簡化版，準確度較低

這兩個是用「比對兩個相近高低點」的簡化邏輯做的，不是嚴謹的圖形辨識演算法，
比前面幾個K線組合型態容易誤判，建議把這兩個訊號的重要性看得比其他訊號低一點。

## 買賣訊號說明對照表（技術分析參考，非投資建議）

| 訊號名稱 | 判斷方式 | 一般解讀 | 使用上要注意的事 |
|---|---|---|---|
| **MA黃金交叉** | 短期均線(MA5)由下往上穿越長期均線(MA20) | 原本偏弱的趨勢可能轉強，常被當作進場買進的參考訊號 | 均線是用過去價格算出來的，訊號一定會落後於實際轉折；在盤整、沒有明顯方向的行情中，均線容易反覆糾纏、頻繁出現真假訊號 |
| **MA死亡交叉** | 短期均線(MA5)由上往下穿越長期均線(MA20) | 原本偏強的趨勢可能轉弱，常被當作出場/減碼的參考訊號 | 同上，且死亡交叉不代表股價一定會持續下跌，仍要搭配整體趨勢判斷 |
| **KD黃金交叉** | K值由下往上穿越D值 | 短期動能轉強，尤其發生在KD值20以下（超賣區）時，訊號較有參考價值 | KD反應快、雜訊也多，如果股價長期單邊上漲或下跌，KD容易「鈍化」卡在高檔或低檔，此時交叉訊號的可靠度會下降 |
| **KD死亡交叉** | K值由上往下穿越D值 | 短期動能轉弱，尤其發生在KD值80以上（超買區）時，訊號較有參考價值 | 同上，且KD屬於震盪型指標，在強勢單邊行情中容易太早出場、錯過後續漲勢 |
| **突破近20日壓力** | 收盤價超過近20個交易日的最高價 | 偏多訊號，代表股價創近期新高 | 建議搭配成交量確認，如果突破時量能沒有明顯放大，可能是「假突破」 |
| **跌破近20日支撐** | 收盤價低於近20個交易日的最低價 | 偏空訊號，代表股價創近期新低 | 同上，建議搭配量能與大盤環境一起判斷，避免單一訊號誤判 |

**重要提醒**：以上都是技術分析中很常見的參考工具，但沒有任何單一指標可以百分之百準確預測漲跌。這些訊號比較適合當作「觀察清單」的篩選依據，實際進出場前建議自己再綜合基本面、籌碼面、大盤環境一起評估，並做好資金控管。本系統呈現的訊號僅供參考，不構成投資建議。

## 這次新增：買賣訊號提醒 + 市場熱度潛力股

### 核心自選股買賣訊號（僅供參考，不構成投資建議）

`monitor.py` 現在會比對「今天 vs 昨天」的技術指標狀態，偵測三種訊號：
- **均線交叉**：MA5 穿越 MA20（黃金交叉／死亡交叉）
- **KD交叉**：K線穿越D線（黃金交叉／死亡交叉）
- **支撐/壓力突破**：收盤價突破近20日高點（偏多）或跌破近20日低點（偏空）

觸發的訊號會：
1. 顯示在 LINE 推播訊息裡（核心自選股名稱下方會多一行 ⚡ 提示）
2. 顯示在網頁的核心自選股卡片下方（紅色徽章＝偏多/買進參考，綠色徽章＝偏空/賣出參考）

資料存在 `docs/data/signals.json`。**因為訊號需要比對「今天 vs 昨天」，剛上線第一天不會有任何訊號，累積至少2天資料後才會開始出現。**

### 市場熱度潛力股

新增一個動態分類「熱度潛力股」，邏輯是：只看當日上漲的股票，綜合「成交金額排名」＋「漲幅排名」算出熱度分數，
分數最好的前10檔視為當日市場關注度最高、動能最強的股票。這個分類不需要額外的歷史資料庫，
每天用當日資料就能算，網頁會自動顯示在「動態觀察清單」區塊裡。

### 之後可以繼續加的（依你之前的優先順序）

- 法人趨勢潛力股：目前只有「當日買賣超排行」，可以再加「連續N天外資買超」的趨勢判斷
- EPS基本面潛力股：需要另外串接證交所的月營收/財報端點，目前還沒開始做

## 這次更新的網頁功能（淺色主題版）

- **配色改為淺色系**：頁面底色改成暖白色，卡片為白底，頂部跑馬燈維持深色招牌條做對比
- **卡片走勢小圖表**：核心自選股卡片下方會顯示近期收盤價走勢的小圖表（sparkline），
  資料來源是 `docs/data/history.json`，由 `monitor.py` 每次執行時自動累積（目前保留約半年歷史）
  ***剛上線的前幾天會顯示「尚無多日歷史資料」，這是正常現象，累積幾天後就會出現線圖***
- **今日新聞區塊**：串接證交所 OpenAPI 的「重大訊息」端點（`t187ap04_L`，就是公開資訊觀測站的資料來源），
  免費、不需要金鑰，顯示在網頁最下方，資料存在 `docs/data/news.csv`
- **版面縮窄至85%寬度**：桌機瀏覽器下內容區域維持在螢幕的85%寬度置中顯示，文字大小不受影響；
  手機瀏覽器因螢幕本身較窄，改為92%寬度，避免內容太擠
- **跑馬燈速度放慢**：捲動一輪時間從40秒延長到90秒
- **手機文字放大**：手機瀏覽器下的股價、跑馬燈、新聞標題字級都比之前放大
